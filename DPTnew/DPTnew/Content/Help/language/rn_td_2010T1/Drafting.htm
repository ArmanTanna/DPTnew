New Drafting & Dimension Functionalities and Improvements

Version 2010.1 introduces the following enhancements to the Drafting & Dimension functionalities.

  • Automatic Dimensioning for an Electrode
  • New layers and view attributes for Bend Data of a Flat Pattern View
  • Input of ASCII file with only 2 coordinates

Automatic Dimensioning for an Electrode

In the drawing environment, you can create dimensions between the electrode reference point and the Work Plane origin. The dimensions represent the X and Y displacement of the reference point from the origin. These X and Y dimensions are created with respect to the vertical and horizontal reference axes displayed from the origin. To achieve this goal, the Enable for electrode origin check box and Axis length text box have been introduced in the Document Properties dialog box.

Note
  • The reference points are specified while creating an Electrode.
  • During the process of automatic dimensioning, a reference axes are created automatically from the electrode origin only when the Work Plane during electrode creation is set such that the XY, YZ or XZ planes are parallel to the view plane.

New layers and view attributes for Bend Data of a Flat Pattern View

Significant enhancements have been made in the Flat Pattern category of the Document Properties and Entity Properties dialog boxes. They enable you to:

  • Customize the view attributes of a bend and bend table.
  • Create bend lines, bend data, and bend table for each flat pattern view on different layers.
  • Generate a bend table while creating a flat pattern view.

Input of ASCII file with only 2 coordinates

For the Drawing commands that take an input of an ASCII file, you can use one that contains point coordinate information for the X and Y coordinates only. In case the ASCII file contains Z-coordinate information, it is ignored.
