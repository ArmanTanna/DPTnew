DXF/DWG

Category: Load

Load

The controls in this area determine how Model and Paper spaces are to be converted.

Spaces

The controls in this area determine how Model and Paper spaces are to be converted.

Model and paper spaces

Enables you to load both the Model and Paper space. If the DWG or DXF file contains both Model space and Paper spaces, an alert box is displayed asking if you want to load both the Model space and Paper spaces. If you click Yes, the program creates one drawing containing the Model space, and as many additional drawings as the number of Paper spaces to be converted, each containing a different Paper space.

Model space

Enables you to load only Model Space objects.

Paper spaces

Enables you to load only Paper space objects. The program will create as many drawings as the number of Paper spaces to be converted, each containing a different Paper space.

Blocks

The controls in this group determine how the AutoCAD blocks will be converted.

As shared groups

When selected, blocks are converted to shared groups.

As groups

When selected, blocks are converted to non-shared groups.

Only used blocks

When selected, only the blocks actually used in the drawing being imported are converted. Blocks defined but not used are ignored and the resulting drawing may be smaller in size. This options is enabled when converting blocks as shared groups.

Optimize performances

When checked, an optimization is performed during the conversion, assuming some specific entities are contained inside each block to be converted. For this reason, optimization speeds the process up a good deal, but in some cases it might lead to results differing from the expectations. When cleared, blocks are converted to group entities with no optimization.

Disassemble on loading

The controls in this area determine whether certain entities are converted into their corresponding think3 entities when the file is opened, or broken down into their component entities (such as segments, arcs, and so on).

Dimension/label

When selected, dimensions are broken down into their component entities; when not selected, they are converted into think3 dimension entities.
