Opening files with missing references to component files

When you open a file including some references, typically belonging to an assembly file, which the think3 application is unable to load, the Reference Manager is automatically activated, and the program displays the Fix Missing References and other dialog boxes, from where you can:

  • restore the lost references
  • select a new reference
  • ignore some possible missing reference and load the file.

See Reference Manager for details.

However, if you click the Cancel button in every dialog box proposed by the Reference Manager, a warning message signaling that it's impossible to load one or more components is displayed. You can then choose between the following options:

  • Continue —  continues the loading action without inserting the missing components, which will appear marked by the Delete cross in red in the Model Structure.
  • Close file —  closes the loading action and ends the Open command.
