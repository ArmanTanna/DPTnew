The Assign Part Number command enables you to assign the part data to a part in the selected thinkparts catalog.

You can find this command in the context menu that becomes available when you right-click on a part in an open catalog. Your thinkPLM administrator may have disabled this function for your thinkPLM account, role or group.

Upon execution, thinkparts displays the Part Data dialog as defined for the future type of the selected part. This enables you to enter the data relevant to the selected component as defined in your thinkPLM configuration. Besides the standard tabs, the dialog contains one or more thinkparts tabs that enable you to inspect the part values defined.

Once the part number has been assigned to a part, the latter is inserted as a component in your thinkPLM database. A new entry is also added to the thinkparts database with the codified data; this enables you to use the part to generate other components with different part numbers.

The data relevant to the codified components can be exported to remote installations of thinkparts where they can be imported, provided the same thinkparts catalog exists.
