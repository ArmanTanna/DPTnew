Playback

The Playback command enables you to play back the actions previously recorded to the current log file or Play list (a list of log files with a ".lst" file extension).

To playback select the log file or play list file:

  1. Select Open to display the Capture & Playback file dialog box.



  2. Select the log file, and then click Open.

  3. Once the log file is opened, then all the options in the Capture & Playback dialog box becomes active. The following are the different options available for selection.


Behavior of commands used to save documents during a Playback session

Commands used to save documents (such as Save, Save As, Partial Save or Make X-Ref) during a Playback session don't save documents to the original folder (the one the document was saved to during the Capture session), but rather to an auxiliary folder, which is automatically removed during Playback.

This behavior:

  • Enables the program to play back the log file on a work station where the directory structure is different from the one used when capturing (or even on the same work station, if the original directory structure was modified).
  • Prevents the program from accidentally overwriting documents when performing a Playback session.

The behavior of the Playback function depends on the status of the various Playback options. For a detailed description of these options, read CP - Options and Tools.

Note that if a document is open when you start any of the Playback commands, the following warning message is displayed "All documents will be closed. This will cause the loss of changes. Are you sure you want to start playback on file 'C:\Models\logfile1.cp3'?"

Make sure you have saved all the documents that might be open before proceeding, because once the Playback session starts, all open files will be closed. After you've saved the files, click Yes to start the Playback session.
