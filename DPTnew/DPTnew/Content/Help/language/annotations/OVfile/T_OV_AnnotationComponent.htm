Annotations in Assemblies

Annotations can be used profitably to attach information to assemblies. This documents supplies some tips about how to manage annotations within assemblies.
 

  • The basic rule
    The behavior of annotations in assemblies revolves around the following simple rule: Annotations can exist either at the top-level of an assembly or within a component. The assembly level at which an annotation exists is determined by which component is set as current at the time of its creation. If no component is current when the annotation is created, then it will exist at the top-level. Annotations are visible and accessible only when their container component is set to current. Once you are aware of this basic rule, you can go on reading the document.
     
  • Including annotations when creating a component
    When you create a component (see New Component), annotations already existing in the model and which you want to be associated with the component must be selected along with the other objects to be included in the component. If you don't explicitly select the annotations, although their point of interest can belong to an object which will be included in the component, they will not be added to the component and will remain at the top level.
     
  • Viewing annotations
    When a component is created from a solid and annotations, the latter will not be visible, unless you set the component as current.

    Component set as current: annotations visible
    Component not set as current: annotations not visible

    In other words, by default only annotations at the top-level of the document are visible. When a component is set as current only those annotations contained in it are displayed.
     
  • Inserting annotations
    According to the above mentioned basic rule, annotations are generally added to entities at the top-level of the document. If a component is set as current, you can add the annotation to the component specifying a point belonging to an entity within the component as the Point of Interest.
     
  • Editing and deleting annotations
    Once more, according to the basic rule, you will be enabled to edit/delete all the annotations at the top level, if no component is set as current. You will be able to delete only the annotations belonging to a component, if the latter is et as current.
     
  • Annotation Tree Panel
    The Annotation Tree Panel will update its contents to show annotations that can currently be accessed. By default, the tree will show all annotations at the top-level. If a component is set as current, the Annotation Tree Panel will update its contents to display only the annotations in the current component. The label of the tree's root node will display the name of the current component or the name of the document (if top-level).
