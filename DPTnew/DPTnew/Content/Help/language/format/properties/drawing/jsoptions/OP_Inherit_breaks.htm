When the check box is selected, you are enabled to inherit breaks defined on the main view onto the derived view. This action grays out the Align to main option.
When cleared, cannot inherit breaks from the main view whereas it is possible to align the view to the main view as it activates the Align to main option.


When the option is activate:

  • while creating the derived view if the direction of the break is parallel to the derived view's sight direction, the derived view inherits the multiple breaks from the main view.
  • the changes (edit, delete, add) on the main view multiple breaks are automatically reflected on its derived views.
  • it is not possible to add/delete/modify the breaks on the derived views.

For every regeneration of a derived view the breaks from the main view, if present, are inherited based on the same conditions as defined during the view creation.


Note
Only the multiple breaks from main view can be inherited on the derived views. Half and partial breaks are ignored.
