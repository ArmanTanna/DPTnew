New Compensator Functionalities and Improvements

Version 2009.1 introduces the following enhancements in the Compensator commands.

  • Enhancement in the Process Compensator command
  • Enhancement in the Extract Zones command
  • Enhancement in the Clean Displacement Field command
  • Enhancement in the Compute Distances command

Enhancement in the Process Compensator command

In this version, the Iterations with ratio check box has been introduced under More Options in the Process Compensator selection list.

  • When the check box is selected, you are enabled to specify a scale factor for each stage of the deformation process. For every iteration level, the ratio can be specified in the Ratio at iteration edit box.
  • When the check box is cleared, scale factor cannot be specified.

If Max. No. of Iterations is more than 5, the inputs are split into several ranges. The required range can be selected from the Iterations drop-down list.


Enhancement in the Extract Zones command

In the Extract Zones command, for the Mode set as Curvature difference %, it is required to select First Points and Second Points set of inputs between which the difference would be calculated. In this context, the following options have been renamed in this release.

Old name New name
Leading Points First Points
Leaded Points Second Points
Leading Surfaces First surfaces

 

The Second surfaces node has been introduced to select the reference surface for the second set of input points. Also, it is mandatory to make selection for First surfaces and Second surfaces for ASCII file and Standalone points as the criterion of computation is based on the differences of the two.


Also the Max. Supposed Distance option has been renamed to Max. presumed distance.


Enhancement in the Clean Displacement Field command

In this version, the new No. loops option has been introduced in the Clean Displacement Field selection list. It enables you to run the filtration process a specified number of times, to remove points progressively.


When the command is applied for the first time some points are removed according to the chosen criteria. This changes the neighborhood and average computations, so by running the filtration loop again, the program could find again some others points to remove.


Enhancement in the Compute Distances command

In this version, the new Max. presumed distance check box has been introduced in the Compute Distances selection list.

  • When the check box is selected, the system discards the distance-to-shape computation of points that are farther than the specified threshold value. Hence, there is significant reduction in the computation time.
  • When the check box is cleared, all the points are included in the computation.
