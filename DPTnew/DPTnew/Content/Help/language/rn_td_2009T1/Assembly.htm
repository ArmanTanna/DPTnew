New Assembly Functionalities and Improvements

Version 2009.1 has introduced the following enhancements to the Assembly functionalities.

  • Improved X-Reference command
  • Enhancement in the View Exploded Assembly command
  • Enhancement in Model Structure Representation of Components
  • Enhancement in Drag and Drop behavior

Improved X-Reference command

Starting from this version, you are enabled to simultaneously place multiple instances of a component in an assembly. To facilitate this enhancement, the Insert X-Reference selection list has been introduced.

You can additionally define the point of insertion as the centre of the bounding box by selecting the Move bounding box center under cursor check box.

Enhancement in the View Exploded Assembly command

With the enhancements made in the View Exploded Assembly command, it is possible to achieve smooth transition of parts from their initial to their final position during the explode sequence. The frame rate for the transition is to be specified in the new Frames per step drop-down list introduced in the View Exploded Assembly selection list.

The other significant enhancements are in capturing images of the explode process. The Capture images check box has been replaced with the new Create image(s) node. Using the options under it you can:

  1. Create images in different formats like BMP, JPG, TIF, GIF etc.
  2. Use the Mode drop-down list to create:
    • a single image at the current step
    • images for a set of steps
    • images for the whole sequence
  3. Define the image resolution in the Image resolution drop-down list.

Select the new Create Images button () to create the bitmaps while the explode sequence is being played.


Enhancement in Model Structure Representation of Components

From this release, a new option Mark as Work in Progress is available on the context menu of the component in the history tree. It highlights the component in Red color in the history tree so that those components can be identified easily in the current session. This color representation is especially helpful in large assemblies having multiple components. To disable the color of the component you can select Unmark as Work in Progress from the context menu.


Enhancement in Drag and Drop behavior

Henceforth, the positioned or constrained parts when moved or dragged are visualized in Shaded View instead of the Wireframe View.
