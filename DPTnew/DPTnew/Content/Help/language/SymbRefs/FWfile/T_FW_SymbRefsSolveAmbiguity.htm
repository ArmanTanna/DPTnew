ModifySymbolic ReferenceSolve Ambiguities


 

Any time you get ambiguities involving Symbolic References, you need to solve them.

Typically, these situations occur when a component provides Symbolic References in an ambiguous way or in a mating event, when, replacing a component, you require to keep the event and unique Symbolic References are not solved automatically by the think3 application.

The Solve Ambiguities command is designed for this specific aim: from the selection list displayed, you are enabled to select the component where ambiguities, involving Symbolic References, occur and specify the Symbolic Reference to be actually adopted.

You also view the information on the event where ambiguities occur and the name of the Symbolic Reference involved.

Note that the command is also started whenever ambiguities involving Symbolic References arise (for instance, when you replace a component involved in one or more mating events and you decide to keep the event(s), some ambiguities involving Symbolic References occur, you are asked to solve them on-line and you accept).
