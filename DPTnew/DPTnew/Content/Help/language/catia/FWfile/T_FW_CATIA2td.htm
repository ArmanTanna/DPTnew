The From CATIA to command loads a CATIA® model file into a new document.

You can find this command in the TD Catia v4 interface toolbar. The command is active both in the 3D and 2D environments of this software. To activate the toolbar, please refer to the TD Catia v4 interface installation procedure.

When you run the command, the file selection dialog box is displayed, from which you can select an existing CATIA® Model or Export file. TD Catia v4 interface narrows automatically the list of files displayed in the selected location to those having extension .model, .dlv or .exp.

Note that a .exp file is actually a collection of models. If you open such a file, a dialog is displayed with the list of models present in the .exp file. This enables you to open the models one at a time.

Once you open a valid CATIA® file, a new model window is opened and the geometry loads into it.

In order to define the parameters controlling the conversion of entities, set the TD Catia v4 interface Options relative to the command before applying it; double check these options if the result of the conversion is not what you expected.

Depending upon your workstation performance and the model complexity, the conversion may require some time to complete. To have an idea of the ongoing of the operation, you may want to open the I/O Window, if concealed, where the steps of the conversion process are displayed.

When you load a CATIA® file, a log file of the conversion is created in the same directory of the file you loaded with the same name of the file you saved and the ".log" extension. Similarly, a file with the same name but ".err" extension will list any error that may occur during the conversion.

Known limitations
  • If the Unit of Measure of the CATIA® file is different from the one currently in use, then while opening that CATIA® file, a warning dialog box is displayed informing you that the Unit of Measure will be changed to the one of the CATIA® file. Subsequently, the current Unit of Measure of the application is changed (not converted) to the one of the CATIA® file.
  • The CATIA® entities can be converted only if the file is created working with Draw Elements; if the CATIA® file is created using transparency in AuxView and AuxView 2, the CATIA® entities can't be converted.
  • Special characters may lead to incorrect conversion.
  • Geometry placement may lead to incorrect conversion.
  • Text positioning may lead to incorrect conversion.
  • Color mapping may lead to incorrect conversion.
  • More warnings and limitations are listed in the Conversion Table for importing from CATIA and limitations (3d entities) and Conversion Table for importing from CATIA (2d entities).
