Rubberbanding mode

Rubberbanding mode:

Select the rendering engine to be used for rubberbanding functions.

Display mode

Display mode:

Enables you to choose the way previews are to be displayed in commands:

  • Wireframe — previews are displayed in wireframe mode.
  • Opaque — all previews are rendered as opaque.
  • Transparent — all previews are rendered as transparent.

A Default transparency factor box is also available to define the transparency factor when the Preview mode is set to Transparent.

Real-time display mode

Tessellation parameters:

Enables you to select the way surfaces must be represented while you are creating/modifying them:

Default The surface being created/modified is represented together (that is 'in place') with the original one in the model.
Simplified The surface being created/modified is represented as not really included in the model, but as 'in face/on top' of it: this is a much faster mode.

Tessellation parameters

Tessellation parameters: Enables you to manage the preview quality.

Default Surface previews will be displayed using the default 0.15 value. This choice will speed up automatic previews in all commands where they are activated, though the preview quality might differ from the final result of the command (for example, zebra lines might look jagged in the preview), for which the current tessellation settings will however be used.
Current Surface previews will be displayed using the current document values, as set by the user. In this case the quality of previews will be higher, though the time required for computation might be longer.
