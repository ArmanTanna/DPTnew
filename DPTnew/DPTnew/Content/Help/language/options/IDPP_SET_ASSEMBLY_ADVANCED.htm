References

References

The controls in this area enables you to control the way paths of the inserted shared groups are stored (relative or absolute):

  • Control the way paths to files containing assembly X-Reference components and annotations are stored (relative or absolute)
  • Control the behavior of X-Reference components when they are modified in their own original file.

Keep absolute file path

Keep absolute file path

When selected, component files of an assembly and external files belonging to annotations are referenced using their absolute file path.

If this check box is not selected, the program stores the file path in a relative form, in which case it may not be valid for all the computers on the network.

Show component/shared group name in structure

Show component/shared group name in structure

When selected, the component/shared group name defined in the General category of the Document Properties (if any) will be displayed in the Model Structure/ Drawing Structure Drawing Structure within brackets on the right of the model/drawing name.

When cleared, the component/shared group name will not be displayed.

Update mode:

Update mode:

This option enables you to control the behavior of X-Reference components when they are modified in their parent file:

Always reload automatically reloaded into any open model in which they are referenced after any change has been made on them. This is the default option.
Never reload never automatically reloaded (if this option is set, after an X-reference component has been modified in its parent file, you need to use the Reload Component command to see it modified also in the open models in which they are referenced).
Ask user confirmation request will be issued before reloading a modified X-Reference component.
