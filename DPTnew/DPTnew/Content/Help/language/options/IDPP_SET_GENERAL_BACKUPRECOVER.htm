System Options

Category: General-Backup/Recover

AutoSave

Frequency (min.):

In order to prevent your work from accidentally spoiling and to enable you to save anything you've done during the session, an automatic saving mechanism is available: if you haven't saved your changes by the time a specified time period has elapsed, the program automatically saves the current models/drawings in each active window.

In the box, enter the value which will determine the interval, measured in minutes, between successive automatic saves.


Actual usefulness of the auto-saving mechanism
The auto-saving mechanism can be very useful whenever the program should terminate abnormally.
In such situations, when you restart the application, the document that was automatically saved — which contains all the changes you made since when you opened or first saved it up to the very last time it was automatically saved — can be automatically recovered by the program itself.
Please note that if the time elapsed since when you opened or first saved your document to when the program stopped abnormally was shorter than the auto-saving frequency range, your original document — with no changes — will be recovered instead.


Mechanism NOT active when the Vault is involved
The auto-saving mechanism is not available — that is: no automatic save action is performed, since there is no automatic indication about where the document should be saved among all the available repositories — if the vaulting saving method is enabled (see thinkPLM integration customization). The vaulting system consists in adding different repositories that can be useful in order to distribute the load (number of files) so as to avoid excessive retrieval times (Windows slows down a lot when a folder contains too many files), or in order to physically separate files that pertain to different projects.

AutoBackup

Limit backup folder size to

The options under the AutoBackup area are explicitly developed to uphold the backup and clearout policies.

Enable

Enable Auto Backup

This check box enables/disables the options under the AutoBackup area.

Max. number of backup copies

Maximum number of backup copies

This option is endowed to manage the number of backup copies for each Model/Drawing file. Each back up copy have suffix .bak followed by the back up number e.g. Coupling.e3.bak.1, Coupling.e3.bak.2. You need to enter the number of backup copies in the edit box required to save.

Backup folder location

Delete backup files older than

The two radio buttons under the Backup folder location area enables you to customize the location of the back up files to be saved.

Same as document folder

Recover/Undo Recover Same as document folder

It enables you to save the back up files in the folder where original e3/e2 files are saved.

Other folders

Other folder

The Other folders radio button enables you to customize the location of the back up files to be saved. Click on the to display the selection list box and select the new location of the back up file to be saved. The location gets display in the edit box.

Cleanup

CleanUP policy

The three options in the Cleanup drop-down list maintains the cleanout of the back up files.
The options are:

  • Never delete backup: It does not delete any backup file from the back up file stored location.
  • Limit backup folder size to: You can enter the memory size of the folder (in which the backup files are stored) in the MB edit box. It enables you to retain the files of the size specified in the MB edit box.
  • Delete backup files older than: It enables you to maintain the back up files on the basis of the number of days specified in the days edit box.
