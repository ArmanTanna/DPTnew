System Options

Category: Profile

General

General

The controls in this area determine the behavior of the program when modifying constrained entities.

Automatic resolving

Automatic resolving

Controls the automatic resolving of profiles following changes in dimensional constraints values. When selected, a profile is automatically updated each time one of its dimensional constraints is changed.
When not selected, you must explicitly update the profile after modifying dimensional constraints (see Update Profile).

3D Reference system

3D Reference system

The 3D profiles are at all times referenced to the Profile Reference System.


  • When the 3D Reference system check box is selected, the Profile Reference System is displayed in the 3D Profile environment.
  • When this check box is cleared, the Profile Reference System is not displayed.

View on Work Plane

View on Work Plane

When selected, the program automatically aligns the view with the Work Plane as soon as you start editing a profile.
When not selected, the view is not aligned to the Work Plane.

Automatic update

Automatic update

When selected and a profile is updated following the modification of more or more of its dimensional constraints (see Update Profile), the entities dependent on the profile (such as extruded solids and features) are also automatically updated.
When not selected, related entities are not automatically updated.

Add ground constraint to origin

Add ground constraint to origin

Enables you to determine whether to automatically add a Ground constraint on the origin of the XY axes or not.

  • If the box is checked, a Ground constraint will be added on the origin of the XY axes.
  • If the box is cleared, no Ground constraint will be added on the origin of the XY axes.

Overdefined profiles

Overdefined profiles:

A profile is overdefined if the number of constraints applied to it is higher than its degrees of freedom.

The Overdefined profiles: drop-down list enables you to determine the behavior of the program with regard to overdefined profiles:

Forbidden Overdefined profiles are not allowed.
Ask user The program will explicitly ask you the action to proceed, each time it encounters an overdefined profile.
Allowed The program allows the presence of overdefined profiles.

Automatic constraints

Automatic constraints

The controls in this area enable you to automatically apply any of the active constraints (i.e.: any constraint for which the corresponding check box is selected). Constraints can be applied whenever the Profile environment is active.

Enable in profile creation

Enable in profile creation

When selected, the appropriate active constraints are applied automatically when you create a profile in the Preselection mode.
When not selected, the appropriate active constraints are applied when you select the AutoConstraint command.

Ground

Ground

When selected, the program automatically assigns the Ground Constraint to one of the following points:

  • The first endpoint (when the profile doesn't contain an arc).
  • The center of the first arc selected (when the profile contains arcs).

Radius

Radius

When selected, the program automatically assigns a dimensional arc radius constraint to each arc or circle.

Orientation

Orientation

When selected, the program automatically assigns a horizontal or vertical constraint to any line whose angle from vertical or horizontal varies less than the value specified in the Angular tolerance.

Perpendicular

Perpendicular

When selected, the program automatically assigns a Perpendicularity Constraint to any pair of lines meeting at a 90-degree angle (plus or minus the value specified in the Angular tolerance box).

Tangent

Tangent

When selected, the program automatically assigns a Tangency Constraint to contiguous lines, arcs and curves.

Coincident

Coincident

When selected, the program automatically assigns a Coincidence Constraint to the centers of arcs and circles that are concentric (within the value specified in the Linear tolerance box).

Tolerance

Tolerance

The controls in this area determine the angular and linear variation within which the program automatically applies constraints.

Angular

Angular:

Determines the tolerance for the horizontal, vertical and perpendicular direction constraints. In other words, this value is the one used to determine whether the automatic Orientation and Perpendicular constraints — when enabled —  are to be applied or not.

Linear

Linear:

Determines the tolerance for Coincidence constraints. In other words, this value is the one used to determine whether the automatic Coincident constraint — when enabled —  is to be applied or not.
