System Options

Category: Dynamic Visualization

Dynamic visualization

Dynamic visualization

The controls in this area determine the dynamic visualization of views when they are dragged (to rotate) using the mouse.

Automatic mode

Automatic mode

When this radio button is selected, the correspondig options become active. The settings made for these options will be applied by the program to automatically adapt the detail level of the visualization in the dynamic movements (zoom / pan / rotate) to get the best performance depending on the current model and system configuration.

Max. pre-processing latency

sec.

In the edit box specify the maximum time you want to wait before the movement of the view starts. This time is used to build the hardware display list that provides a better speed in the movements. If the system can't build the display list within this time, it will try to build a less detailed display list in the following sequence.

  1. Full display list with no simplification
  2. Full display list with some simplification
  3. Simplified display list with the same simplification as the previous step
  4. Simplified display list with a higher simplification.

The program tries to process each step within the time (seconds) specified in the Max. pre-processing latency: edit box to build the display list. If it fails, it tries the next (more simplified) step.

Frame rate

fps

In the edit box, specify the framerate at which the user wants to move the view when the construction of the display list fails. If the software is not able to create the hardware list within the time (seconds) specified in the Max. pre-processing latency: edit box, then it will dynamically change the simplification level and display the maximum possible details such that the specified frame rate can be achieved.

Manual mode

Manual mode

When this radio button is selected, the correspondig options become active. Using these options you can manually make the settings to set the detail level of the visualization in the dynamic movements (zoom / pan rotate).

Hardware display list

Hardware display list:

The program uses the display list buffer on the graphics card (i.e.:the hardware display list), if available, rather than storing graphic entities in its own software display list, to improve performance, particularly when rotating the model.
Choose the hardware display list mode from the drop-down list.

Simplified Use a simplified hardware display list (faster).
Full Use a complete hardware display list (more accurate).

Quality/speed

Quality/speed

When checked, it enables the detail level slider control.
The slider position determines whether to get better modeldrawing visualization quality (with slower rotationpanning speed) or faster speed of rotationpanning (with poor visualization quality).

Full detail

Full detail

All details — i.e.: all objects, no matter how small they are — will be displayed during panning, zooming and rotations of the model. When you set the slider to Full detail, during dynamic movements, the graphics quality is not reduced. That is, the program will automatically set the graphics at a higher quality possible. This may result in a drop in performance , in the case of heavy models.

See the description of the slider for further details.

Fast

Fast

Details — i.e: small objects — will be displayed using a point rather than their real representation during panning, zooming and rotations of the model.

When the slider is pushed to Fast, the graphics quality is reduced during translations, resulting in a better performance, even with heavy models files. Of course, after the translation operation the model drawing is displayed in full quality.

See the description of the slider for further details.
