New Assembly Functionalities and Improvements

Version 2008.1 introduces the following improvements to the Assembly functionalities.

  • Mating Feedback Improvements
  • Mating failure by conflicting constraints

Mating Feedback Improvements

In this version, many visualization improvements have been made in the representation of the mating events, both in the Graphics Area and Model Structure.

Improvement in mating visualization in the Graphics Area


The labels that are displayed upon selecting a mating event is now color coded.


Green : The mating constraint is consistent
Yellow : The mating constraint is having dangling references
Red : The mating constraint is conflicting



There have been improvements in the way the constraints are represented in the Graphics Area; X marker (with solid line) and O marker (with dashed line) are displayed over the positioned and referenced geometry respectively.


Improvement in mating visualization in the Model Structure

Specific icons in the Model Structure represent that the mating constraint is not consistent.

- Mating failure
- Conflicting constraint
- Missing constraint references


Mating failure by conflicting constraints

In previous versions, the conflicting constraints only caused a warning to appear without the failure of the mating event itself. With the enhancements in this version, the mating event now completely fails in such situations and is highlighted by the failed event icon () in the Model Structure. Also the Warning dialog box is displayed where you can see the details about the failure.
