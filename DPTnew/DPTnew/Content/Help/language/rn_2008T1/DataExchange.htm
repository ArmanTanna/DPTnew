Enhancements in the DXF/DWG Converter

The following DXF/DWG converter enhancements have been made in this version.

Entity support for the DXF/DWG format

From this version onwards the ability of the think3 application to support AutoCAD® entities has been augmented. As a result, the entities listed in the following table are now correctly mapped.


AutoCAD® think3 Conversion
Region Group Load
Table Group Load
Line Weight Line Width Load and Save
External Reference X-ref Load and Save
Raster Image reference Image Load and Save
Oblique Dimension Group Load
Leader Dimension with Spline leader line Group Load

Refer to the DXF/DWG to think3 conversion table and think3 to DXF/DWG conversion table in the Online Documentation for details.

Enhancements in the Pro/ENGINEER® Converter

The following enhancements have been made in this version for the Pro/ENGINEER® converter.

Enhancement in the Save category

A new Convert solid entities to components check box has been added to the Save category of the ProEngineer Options dialog box.



  • When selected, the individual solids in the model will be saved as individual Parts in Pro/ENGINEER®.
  • When cleared, all the solids in the model will be saved as a single Solid in Pro/ENGINEER®.

Latest version support

With the improvements made in this version, TD Direct is now able to open Pro/ENGINEER® model files up to version Wildfire 3.0.

Enhancements in the STEP Converter

The following enhancements have been made in this version for the STEP converter.

Reorganized STEP categories

From this version onwards the various categories present in the STEP Options dialog box have been reorganized as shown below.



The old Settings category has been removed and the two options previously present in it are now available in the new General category. Note that the old Convert non-visible entities option has been renamed as Convert hidden entities.


Entity Name support for STEP format

The think3 applications now support reading and writing entity names to and from the STEP format. To save an entity name to a STEP file, you need to create a User Defined Class (by using the Create UDC command) with its Name as TD_ENTITY_NAME and the Type set to Character.



Refer to "Insert a new class of user defined properties" in the online documentation for details on how to insert a User Defined Class.


For the specified class, using the Assign UDP command assign the User Defined Property to any entity (like solid, surfaces or curves) and then set its value as the name to be allocated to the entity for the STEP conversion.



Refer to "Assign user defined properties to entity" in the online documentation for details on how to specify a value to the User Defined Property.


If you save the solid to a STEP file, then the entity will be saved with the name property also applied to it. Similarly, if you try to load a STEP file containing an entity which the name is applied to, the same name will be assigned to the think3 entity as the property value for the TD_ENTITY_NAME class.


New General - Layers category

With the significant enhancements done in the Layers functionality, think3 applications now support assigning alphanumeric values (strings) as layer names. However, STEP being a neutral format, manages layer names only in numeric format. Therefore, it becomes necessary to correctly map the layer information while converting to and from the think3 format to the STEP format. This mapping process can be controlled by the options of the new General - Layers category of the STEP Options dialog box.


Enhancement in the Load - Advanced category

A new Split closed faces check box has been added in the Load - Advanced category.



  • When selected, the closed faces of solids (not surfaces) like cylindrical faces and rotational faces will be split before loading in think3.
  • When cleared, no splitting of closed faces of solids will take place.
