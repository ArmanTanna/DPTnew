Enhancements in the System Options

Enhancement in the General category

In this version, the new 3D Converters area has been introduced in the General category. The option in this area is used to perform quality checks on the model loaded using the 3D converters. As a result, you can view any topological or geometric defects present in the model.



  • If the Check Model Quality after loading check box is selected, after loading the model using the converter, the program automatically starts and applies the Model Quality command on it to perform the quality check.

    Note
    The quality check is done only for some supported formats; these are STEP, IGES, VDA, Parasolid, CATIA V5 and ProEngineer.

  • If the check box is cleared, no quality check is performed on the loaded model.

The new General-Advanced category

A new General-Advanced category has been created in this version.



The controls in the Virtual memory area enable you to achieve complete control over the usage of virtual memory by the application. You can activate the display of warning when the remaining free virtual memory reaches a threshold value, which you can freely define in the Minimum size (MB) edit box.

 

The Show tabs Model, 2D Profile and 3D Profile check box under Next model document enables you to control the visualization of the specific tabs under the Graphics Area for the Model, 2D and 3D profile environment.


  • When the check box is selected, the tabs are displayed and available.
  • When the check box is cleared, no tabs are displayed.

 

The new option Show default prefix in new layer names under Layers in the General-Advanced category of the System Options enables you to add the prefix Layer to the name of the new layer.


The changes take effect only when you open the next document.

Enhancement in the Input category

The new Set direction/axes using X, Y, Z check box has been introduced in this category.



  • When the check box is selected, the direction/axis definition drop-down list will include the X, Y, Z (Parallel to X and through point, Parallel to Y and through point, Parallel to Z and through point) options.
  • When the check box is cleared, and if the program requires the definition of an axis or a direction, you can define the direction as parallel to one of the axes of the Work Plane just by choosing the Line option and by clicking on one of them.

Refer to the Enhancements in the general method for axis, line and direction definition topic for details.


The new Reference Systems category

The options in this new category control the display of the Work Plane and the Absolute Reference System (World). Jointly these two systems are called as the Reference Systems.



You can define whether to display the Reference System at its actual location or as a dummy (Proxy) at the corner of the Graphics Area. This will be helpful in knowing the orientation of the view when the actual Reference System is beyond the visible portion of the Graphics Area.


The new Reference Systems - Appearance category

The options in this new category control the appearance of the Reference Systems.



Here you can define the way the Reference System is displayed on the Graphics Area in terms of its size, position and constitution. The options in the Axes area are also applicable to the Profile Reference System.


Enhancement in the Profile category

The old Reference System check box has now been replaced with the 3D Reference system check box.



  • When the 3D Reference system check box is selected, the Profile Reference System is displayed in the 3D Profile environment.
  • When this check box is cleared, the Profile Reference System is not displayed.

The new Rendering category

The options in this category controls the display and saving of the rendered images.



Here you can define where the rendered model is displayed or its image is saved.

Removal of the Work Plane category

The Work Plane category present in the previous versions has been removed henceforth. The options in this category were used to control the display and appearance of the Work Plane. Now these controls have been integrated into the options of the new System Options - Reference Systems and System Options - Reference Systems - Appearance categories.


The New Toolbar Object command

The New Toolbar Object command has been introduced in order to enable you to have full control of the customization of existing Dimension Properties toolbars and to enable you to create new ones.

You can find this command in the context menu of the properties of the Dimension and Text categories of the Entity Properties. Refer to the online documentation for a detailed description of the New Toolbar Object command.


Enhancements in the Entity Properties

Enhancements in the General category

The following user interface enhancements have been made in this category:

  1. From now onwards, the associated attribute of the current layer is indicated with the name By layer. This has been made available in all the layer attributes (Color, Line type and Width)
  2. Previously you could specify a layer in the Entity Properties - General category by typing the corresponding number in a text box. Now this box has been replaced with an editable drop-down list in which you can either choose an existing layer or type in the name of a new one to create it.




Enhancements in the Drawing View - View Attributes category

In previous version you could specify a layer by typing the corresponding number in a box. The new Hole Table Options dialog box

This version introduces the new Hole Table Options dialog box to control the customization settings of a Hole Table. The properties of a Hole Table can be edited, using the new Hole Table Options dialog box, in two ways:

  • When the Hole Table command is active, the Hole Table properties can be customized using the Options option.
  • When the Hole Table command is not active, the Hole Table properties can be changed using the FormatOrganizeHole Table command.

Once the current settings are saved, they will be available every time you run the command. The saved settings survive the program session. However, for the ModifyDimensionHole Table command, the program loads the configuration last used to create the same Hole Table.
