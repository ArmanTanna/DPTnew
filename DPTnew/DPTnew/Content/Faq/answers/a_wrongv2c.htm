ERROR 55 - HASP_UPDATE_TOO_NEW




It means that you have not applied yet one (or more) of the previous V2C files issued for the same license. It is mandatory that the V2C files are always applied in the same order as they were issued. Therefore, before applying your last V2C file, make sure you have applied - in the righe order! - all previous V2C files issued for the that license.


If you have not collected all your V2C files, please contact pwdrelease@dptcorporate.com to obtain them.



ERROR 64 - HASP_CLONE_DETECTED




It means that you are applying a V2C file to the wrong machine (= not the one which the file was generated for) or that something happened to your machine, in a way that it gets no longer recognized as before.

Please check with your VAR or with DPT the best solution for you case:
a) Find the right machine
b) Driver reinstallation
c) Export operation and reinstallation on a newer C2V
d) Export operation + Validation through Clear Clone (provided by DPT only) and reinstallation on a newer C2V
e) Affidavit, the license cannot be recovered (Clear Clone doesn't work).



ERROR 65 - HASP_UPDATE_ALREADY_ADDED


It means that the V2C (or V2CP) file has been already applied to that specific machine. If any, please ask for the next V2C update.



ERROR 43 - Device Error


Version drivers are not consistent. You need to reinstall Sentinel drivers.



ERROR 18 - KEYID_NOT_FOUND


The Product Key is not present in the Sentinel Keys section of the Sentinel ACC. It seems you need to apply the first V2C file generated for that machine.



ERROR 19 - INVALID_UPDATE_DATA


It seems that the V2C file content got corrupted during the file exchange. Please download the VC2 file once again.



ERROR 48 - VENDORLIB_NOT_FOUND


The Vendor Library haspvlib_110137.dll is missing from the C:\Program Files (x86)\Common Files\Aladdin Shared\HASP folder.
To solve the problem, you can choose one of the following options:
a) reinstall DPT_Licensing
b) reinstall ThinkDesign
c) manually copy&paste the file from a different PC.



ERROR 54 - HASP_UPDATE_TOO_OLD


It seems that the V2C file you are trying to apply is older than those already applied (i.e. already applied but not the last one). Please get the right V2C update file.



ERROR 66 - HASP_INACTIVE


One or more V2C files are invalid or missing from the directory C:\Program Files (x86)\Common Files\SafeNet Sentinel\Sentinel LDK\installed\110137.
Please identify the missing V2C file through the Sentinel ACC > Sentinel Keys section > Actions column, Certificates button.
Please get the right V2C file and apply it to the Update/Attach section of the Sentinel ACC.
