A C2V file is correct when it contains both the SL_AdminMode section and the SL_UserMode one. Here below is an example:



        <?xml version="1.0" encoding="UTF-8" ?>
		<hasp_info>
  		<host_fingerprint type="SL-AdminMode" 
        crc="3913458469">...</host_fingerprint>
  		<host_fingerprint type="SL-UserMode" vendorid="110137" 
        crc="3016243691">...</host_fingerprint>
		</hasp_info>


If an user without Administrator Rights generates a C2V file, it will present only the SL_AdminMode section.



        <?xml version="1.0" encoding="UTF-8" ?>
		<hasp_info></host_fingerprint>
  		<host_fingerprint type="SL-UserMode" vendorid="110137" 
        crc="3016243691">...</host_fingerprint>
		</hasp_info>


If you get a 'wrong' C2V file, try and re-generate it, making sure you have Administrator Rights on the PC. If the problem persists, please open a Case in the Customer Care portal.
