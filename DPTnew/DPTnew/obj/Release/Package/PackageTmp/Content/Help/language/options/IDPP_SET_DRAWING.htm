System Options

Category: Drawing

History

History

The controls in this area enable you to define, for the drawing entities, the parameters related to the history.

Enable logging

Enable logging

If selected, when you add an entity to a view in such a way that it is parametrically linked to the view, the lock symbol will be displayed and the entity will be logged and displayed in the Drawing Structure.

If not selected, the entity will not be logged and displayed in the Drawing Structure and the lock symbol won't be displayed.

Disable the creation of unrelated dimensions

Disable the creation of unrelated dimensions

When selected,the program prevents you from creating dimensions if they are not fully associative (which can happen, for example, if you try to add a dimension related to an entity inside a drawing view different from the current one).

When not selected, you are enabled to add dimensions referring to entities in views different from the current one. But when the model changes, this type of dimensions will not be updated, as it is not completely correlated to the entities it refers to.

Referenced model

Referenced model

The controls in this area enable you to define the parameters related to the reference model.

AutoLoad model

AutoLoad model

When selected, when you load a drawing related to a reference model, its Edit Reference Model is also loaded.

Rebuild drawing after model changes

Rebuild drawing after model changes

When selected, if the Edit Reference Model related to a drawing is changed, the drawing is automatically rebuilt to reflect those changes.

When not selected, if the reference model changes, the drawing will not be automatically updated (to update it, you will have to use the Rebuild Drawing command).

AutoSave model after drawing saving

AutoSave model after drawing saving

When selected, if changes are made to the drawing, its Reference Model is automatically updated and saved as needed.

When not selected, the reference model is updated, but it is your responsibility to save it..

Layout configuration

Layout configuration

The controls in this area enable you to define the parameters related to the Layout Configuration.

File:

File:

Enables you to define the type of the template to be used: ISO, ANSI, GB, JIS or KS.

Shared group drag&&drop

Shared group drag&drop

The options in this area enable you to specify if shared groups inserted into the drawing from a library (see "Inserting a Group from a Library into your Drawing") and the instances of those added to a library by dragging and dropping (see " Adding a Group to a Library") will be:

Local Locally defined: the shared group and the entities that it contains are stored directly in the drawing.
X-Reference Group X-Reference: the shared group is the think3 application's 2D file and the drawing contains a link to this file.

Local

Local

If this option is selected, shared groups inserted into the drawing from a library and the instances of those added to a library by dragging and dropping will be locally defined.

X-Reference

X-Reference

If this option is selected, shared groups inserted into the drawing from a library and the instances of those added to a library by dragging and dropping will be X-References.

Unlink Model References

Create components as groups

If selected, the Unlink Model References command will create a group for each 3D component. Group name will use the component name as shown in Model Structure of the referenced model.

If not selected, the resulting component entities will not be grouped.

Create components as groups

Create components as groups

If selected, the Unlink Model References command will create a group for each 3D component. Group name will use the component name as shown in Model Structure of the referenced model.

If not selected, the resulting component entities will not be grouped.

View

View

The controls in this area enable you to define drawing view regeneration.

Regenerate:

Regenerate:

Enables you to define the type of the regeneration to be used after properties modifications:

Always The views will be always be regenerated after any changes done to the properties.
Never The views are not regenerated after any changes made to the properties. The icon will show that the view needs to be updated. Use Regenerate View or Regenerate Drawing to manually update the view.
Ask User The user will be prompted a question each time the properties are changed and a view regeneration is in order.
