Virtual memory

Virtual memory

The controls in this area enable you to achieve complete control over the usage of virtual memory by the application.
In fact, you can activate or deactivate the display of a message informing you that the remaining free virtual memory reaches a threshold value that you can freely define.

When the display of the message is active and the corresponding warning is actually displayed, you can perform the appropriate actions so as to avoid crashes or standstills, in the following order.

  • Quit all the other applications that are running contemporarily
  • Close other documents that may be simultaneously open in the think3 application
  • Save the current document, stop the application and then start it again.

Show warning when reaching the minimum size

Show warning when reaching the minimum size

Enables you to control the display of a specific warning message when the application is running out of the virtual memory threshold value you can set in the box below.

  • When the check box is selected, upon reaching the threshold value the warning will be displayed, thus enabling you to perform the proper actions, which may be, in this very order, the following:
    • Quit all the other applications that are running contemporarily
    • Close other documents that may be simultaneously open in the think3 application
    • Save the current document, stop the application and then start it again.
    This choice can be very useful especially in critical situations when you are using big models or many documents are open or other applications are running simultaneously.
  • When the check box is not selected, no warning will be displayed upon approaching the lower limit for the virtual memory usage. Use this mode only if you are sure none of the above listed situations applies to your working method.

Minimum size (MB):

Minimum size (MB):

Enables you to define the threshold value for the virtual memory below which a specific warning message will be displayed when the application is reaching the limit, if the check box above is selected.

A suggested threshold value not to be lowered is about 100 MB.

Next model document

Next model document

The controls in this area enable you to manage the visualization and accessibility of the Model, 2D Profile and 3D Profile tabs under the Graphics Area in the next model document that is opened.

Show tabs Model, 2D Profile and 3D Profile

Show tabs Model, 2D Profile and 3D Profile

This check box controls the visualization and accessibility of the Model, 2D Profile and 3D Profile tabs under the Graphics Area.

  • When the box is checked, the tabs are displayed and available.


     
  • When the box is cleared, no tabs are displayed.


Note
The change in the visibility of the tabs takes effect only when the next model document is opened. There will be no effect on the currently open documents.

Layers

Show default prefix in new layer names

This check box controls the display of prefix, Layer, when creating a new layer.

  • When the box is checked, the program appends the prefix — Layer — to the layer name.
  • When the box is cleared, the prefix is not shown .

Log file management

Error Report

The controls in this area allows you to make settings for configuring the Log File Management to send information from your system.

Disabled

Disable Error Reporting

When this radio button is selected, the error traceback log generated when an error occurs during software usage is not reported using software resources.

Note: The unsent error traceback logs can cleared using the Delete Log Files button, which is active only when this option is selected.

Hidden mode

Hidden Mode

When this radio button is selected, the error log generated when an error occurs during software usage is reported automatically on each startup.

Ask before sending

Ask before sending the error report

When this radio button is selected, the error log generated when an error occurs during software usage is reported using the program's Log File Management only on the user's confirmation.

Delete Log Files...

Clear Report

Click this button to erase all the unsent traceback logs from the User folder\ Application Data\think3\thinkdesigndrafting\x.x\files\Tracebacks  folder.

Note: This button is active only when error reporting is disabled.
