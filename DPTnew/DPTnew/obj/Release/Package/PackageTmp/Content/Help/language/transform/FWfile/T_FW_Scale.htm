Edit Scale

The Scale command enables you to homogeneously scale selected entities, or to perform mono-directional scaling (stretching) along one direction. As soon as the command is started, a set of handles looking like a box is displayed:
 

3D environment 2D environment

  • The central handle is the origin of the scale.
    You can move it by dragging it to any position you like. About the default position of the origin, see Default poisition of the origin in Scaling or Stretching Entities.
  • The three two axes represent the three coordinate directions, initially parallel to the axes of the Work Plane coordinate system. The threetwo Scale text boxes (Scale X, and Scale Y, and Scale Z) are displayed in the Graphics Area. You can scale it independently in the X, Y, and ZX and Y directions by entering the corresponding values in the Scale X, and Scale Y and Scale Z text boxes, respectively.
  • The handle on a box vertex enables you to perform homogeneous scaling. You can scale an object by dragging the handle or by typing the scale into the Scale box that shows up when you click on the handle. The box and the entities preview change accordingly to the scaling.
  • (3D environment only): You can make associative transformations using the check box inside More Options.
Note
The command is also available from the Profile environment. It is possible not only to modify the entities inside the profile mode using this command, but also to retain or modify the constraints or dimensions that are applied to keep the changes intact. When an entity is stretched using the command, the dimensions will be updated to reflect the amount of stretch done to the entity. Also, the command will automatically modify the constraints to suit the condition. If the constraints cannot be modified, the command removes the constraints to keep the changes made.>
