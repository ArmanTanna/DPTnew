New Drafting and Dimension Functionalities and Improvements

Version 2009.2 introduces the following enhancements and improvements to the Drafting and Dimension functionalities.

  • Enhancements in the Manual Balloon command
  • Enhancements in the Label command
  • Enhancements in the Automatic Balloon command
  • Enhancements in the Text command

Enhancements in the Manual Balloon command

Many important enhancements have been made in the Manual Balloon command, with the goal of having balloons as a family. The enhancements include:

  • Balloons with double line
  • Stacking of the balloons
  • Modifying a balloon stack
  • Editing the part properties and updating the part list

Balloons with double line

In the Manual Balloon selection list, the old Text option has been replaced with two new options — Text above and Text below — to permit creation of double line balloons.

Stacking of the balloons

A new and very useful possibility is now available in the Manual Balloon command. When the Stack balloons check box is selected, it is possible to stack the balloons in the order of their component selection. On selecting the check box, the drop-down list with following stacking alignment possibilities is activated.

Horizontally Enables you to stack the balloons horizontally. Depending on the slope of the leader line, the direction (left or right) will be determined.
Vertically up Enables you to stack the balloons in the vertically upward direction.
Vertically down Enables you to stack the balloons in the vertically downward direction.

The following are the various stacking functionalities.


  • When only one component is selected, it is mandatory to select an existing balloon for alignment reference. Once the reference balloon for the alignment is selected, the stacked balloon preview will be displayed as per the selected stacking direction. If the selected reference balloon belongs to already stacked balloons, then the balloon:
    • direction will be same as the reference stack direction, and
    • will be placed at the end of the stacked balloons.


  • When more than one component is selected, the stacking starts automatically and a preview will be displayed with the leader line starting from the origin of the first selected component in the array of components.

  • When the AutoOrder check box is selected, the leader line will start from the component having least RELPOS number (item position) and the balloons will be stacked in ascending order of the RELPOS.
    Note that this check box is available only when the Stack balloons check box is selected.

  • When the frame type is Underlining, the serially stacked balloons will be separated by the "V" symbol and the vertically stacked balloons by the "|" (vertical line) symbol. The symbol size attributes can be defined by using the Text and Frame Offset parameters from the Entity Properties - Text - Font & Frame category.

Modifying a balloon stack

The modifications that can be done on a balloon stack include appending and deletion of a balloon.

The new Insert New Balloon command has been added to the context menu, enabling you to append a balloon to the existing balloon stack. If the reference balloon is from a balloon stack, the new balloon will be appended at the end of the same balloon stack. If not it will be added after the reference balloon itself.


You have the possibility of deleting a balloon from a stack by using the generic Delete command. When you try to delete a balloon with leader line from a balloon family, an appropriate warning will be displayed.


A new Remove related part properties check box has been introduced in the Document Properties - Dimension category to delete the related part properties on every balloon deletion.

Editing the part properties and updating the part list

The new Edit related part properties check box has been introduced in the Manual Balloon selection list. When selected, it displays the Edit Part Properties dialog box after the balloon is placed on the drawing. The properties that are modified in the dialog box get automatically updated in the part list, if present in the drawing.


Enhancements in the Label command

In this version, it is possible to create:


  • double line label entities. To facilitate this, the old Text option is replaced with two new options — Text above and Text below — in the Label selection list.

  • labels with out reference. To do so, you need to clear the Remove unrelated dimensions check box in the Document Properties - Dimension category.

Enhancements in the Automatic Balloon command

The ballooning system has been provided with four new alignments - Left, Top, Right, and Bottom, in addition to the available Automatic ballooning behavior. To accomplish this, the user interface has been significantly improved with the following new options under More Options in the Automatic Balloon selection list:

Alignment Enables you to determine the balloon alignment method.
Distance between Balloons Enables you to define the distance to be maintained between two balloons.
For the Left/ Right alignments, the gap is the distance between bounding boxes of two balloons.
For the Top/ Bottom alignments, the gap is the horizontal distance between two balloons in a row. This is the minimum distance possible between two columns of balloons.
Distance from View Boundary Enables you to set the distance that the balloons maintain from the nearest component.
Arrange on multiple rows Allows you to make matrix like balloons or a balloon in a single line when selected. This option is visible only for Top or Bottom alignments.

In the Automatic Balloon selection list, the old Text option has been replaced with two new options — Text above and Text below — to permit creation of double line balloons.


Enhancements in the Text command

The following significant enhancements have been made in the Text command.

  • Multiline text justification in a bounding box
  • New single line text justifications
  • New Redefine Text command

Multiline text justification in a bounding box

Till previous version, the multiline text positioning is possible only with respect to a point. However, in this version it is also possible to align text with respect to a cell (bounding box with four corners). To facilitate both the existing and the new behavior, a new Multiline text position drop-down list has been introduced under More in the Text selection list.

Following are the characteristics of the new modality:


  • After the placement of the multiline text, one origin handle and the 4 bounding box corner handles will be displayed. Depending on the origin justification, the origin handle might override any of the box corners.

  • While dragging the origin handle, the entire text is moved coherently. However, while dragging any of the box corner handles, the box is stretched and text gets wrapped coherently.

New single line text justifications

The following new single line text justifications in between 2 points have been appended to the existing Origin drop-down list.

Fit to left
Fit in the middle
Fit to right

New Redefine Text command

The new Redefine Text enables you to redefine an existing text in a document. To redefine the selected text, right click on it and select the Redefine Text from the context menu.
