The Layers tab enables you to control the visibility of layers in both the model and drawing environments.

By default only layer 0 is there in a new document. When an old model/drawing is loaded into the new version, the actual layers automatically defined in the document will be:

  • Default layer 0, which is always there
  • All layers with at least one entity is laying on
  • Layers with an associated description, no matter whether empty or not
  • Current layer.

The new layer name will be a string composed of Layer and of the number of the original old layer, like, for instance, Layer15. Layer 0 will remain just 0.

Note
Using the option Show default prefix in new layer names under Layers in the General-Advanced category of the System Options, you can add the prefix Layer to the name of the new layer. Therefore, by unchecking the option the default prefix can be removed when using the older version files.

When importing the content of a document through the Insert From File command, all the defined layers will be imported, regardless of the fact there may be entities laying on them or not.

Compatibility
All the customized layers older than version 2008.1 are no more compatible with the current layer management. The renewed layer customization can be applied only to newly created documents.
