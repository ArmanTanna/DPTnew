The Export Partnumbers command exports to a portable file the parts of the selected catalogs to which a part number has been assigned.

You can find this command in the context menu that displays when right-clicking on a catalog or an index in the Catalog Manager browser frame.

When you start the command, the program opens a folder selection box that enables you to specify a folder where to save the output files. Depending on whether you selected a catalog or an index, the program will create one or more files using as file name the Catalog code, displayed in the catalog properties, with extension tppn.
Each file contains the data of a single catalog; if you selected an index, the program navigates the index tree in order to find all catalogs and creates a tppn file for each catalog.
The file contains the data relevant to the codified parts in the catalog; since these are actual components in your thinkPLM database, the file will contain all the data fields specified in your thinkPLM configuration. If the catalog contains no codified parts, the tppn file will only contain the catalog code.
A progress bar provides you with an estimate of the time remaining to completion.

You can then send the tppn files to a remote installation, where they can be imported with the Import Part Numbers command.

The Import Part Numbers command is designed to work only if a catalog with the same name as the original one exists. If you plan to export to a different location your part numbers, use the Export command to export the actual catalog and send it along with the part numbers data.
