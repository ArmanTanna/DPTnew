Template based creation of new documents

Starting from this version the creation of any new document is based on templates, which can be standard or customized ones.


A template is a document, which contains predefined properties, settings and attributes, and some entities. This will be of great help in order to speed up the activities whenever you need to work on models/drawings in a series with the same basic characteristics. See "Document Templates" for more details.

As a consequence of the new templates concept, the word — Template — in old commands about templates has been renamed to Layout Configuration.

This version helps you to have a template with all of the required parameters pre-set, to enable you to work in an efficient, faster and consistent manner.

In previous versions, you have already used templates to begin a new document but only with the provided predefined settings and standard dimensioning methods. However, now you are enabled to use either the standard or your own customized template. Note that each standard template file supplied with the product contains standard settings for dimension/text styles and unit of measure values and in this case there is no difference with the previous version.

The significant enhancements that have been made in the document template management include:

  • Creating a template
  • Choosing a template
  • Settings for the ThinkDesign start up dialog box

Creating a template

Use the new Save as Template command to save the current document as a template.
To create a new template, see "How to customize a document template" in the online documentation. After creating a new template, it will be available in the list of the Current templates options, which can be chosen using the Change button.

Choosing a template

Now onwards you can select a different template rather than the default one by using the revised ThinkDesign dialog box (to open, select the New File button). It includes the new Current templates area and Change button. When you select the Change button, a new Change Templates dialog box is displayed. It enables you to choose between the current template options: Model and/or Drawing. The templates that are specified here are reflected in the text boxes of the corresponding options in the Current templates area.


Enhancement in the Query command

The Query command has been removed from EditSelect in the menu bar. Now, it is available only in the context menu. On starting the command, instead of the Query selection dialog box, you will get a window listing all the underneath entities (surfaces, lines, curves etc) of the model. You can toggle and select among the entities from the listing window.

The alternate way to use the Query command is by selecting the Automatic query selection check box from the System Options- Input category. The enhancement displays the number of entities which are underneath the snapped entity (based on depth order). You will be able to toggle among the entities either by using the Tab or Page Up/Page Down keys. Also, this tool has been enhanced to support all types of display.

Possibility of constraints with the origin of Work Plane

Now, it is possible to apply geometrical constraints like Coincidence, Point on Curve etc. between the origin of the Work Plane and any point of the 2D Profile.

Enhancement in the Model Quality command

Starting from this version, you will be able to search for non-manifold defects in the model. To facilitate this, a new Not Manifold category has been introduced in the Check Parameters dialog box of the Model Quality command. The default settings of the Check Parameters dialog box have also been changed.

In addition, a new Show All Defects/Hide All Defects () button has been introduced in the selection list. It enables/disables the display of the defects on the model in the Graphics Area.


The new Recover and Undo Recover commands

Backup files are created as per the settings made in the Backup/Recover category. In case of data corruption or if for any reason there is a need to go back to a previous version of the file, the new FileRecover command enables you to open these backup copies of the Model/Drawing files by browsing through a file selection box. The new FileUndo Recover command discards the recover operation and restores the situation as saved in the original file (if there is no data corruption).

Enhancement in the Section View command

The following enhancements have been made in the Section View command.

  • The location of the command has been changed to ViewModifyDefine Section View and the XY-plane of the World Coordinate System is set as the default section plane.
  • Now you can set the color of the capped surface to be the same as that of the sectioned model.
  • A new Show section edges check box has been introduced in the selection list.
    • When selected, the section edges are highlighted.
    • When cleared, the section edges are not highlighted.
  • Henceforth, the ViewDisplaySection View command is used to toggle on\off the section view of the model.

Enhancement in the Properties command (File)

The Edit Part Properties tab has been added to the dialog box automatically opened by the FileProperties command. It enables you to quickly edit the part data relevant to the top level component described in the document.


The new Mean Plane command

This version introduces a new Design Aids command called Mean Plane. It enables you to create a plane (surface) considering the mean location of user selected entities, which can be points, curves or meshes.

Enhancement in Adaptive Measure selectability

With the enhancements made in Adaptive Measure (AM), the scope of graphical selections to capture Radius and Diameter AM has increased. It is now possible to graphically select Canal surfaces whose radius is varying both in U and V directions. The radius that remains constant along one of the directions will be linked to the adaptive measure. For example, the fillet applied on an edge of the cylinder is a canal surface, and hence, it can be graphically selected to link its radius/diameter value using AM.


If you select a non canal surface (for example a variable radius fillet) to apply Adaptive Measure (for capturing Radius or Diameter), a suitable warning will be displayed informing you that your selection is invalid.


Canal Surface definition: It is a surface formed as the envelope of a family of spheres whose centers lie on a space curve; for example, a Torus.


Enhancement in Quality Checks and Analysis

A new Curvature signs functionality has been introduced in the following:

  • As the Curvature signs tab in the Quality Checks dialog box, which is available for many Surfacing and GSM commands
  • As the Curvature Signs command in Analysis
  • As the Curvature signs option in the Type drop-down list of the Automatic analysis command

The Curvature signs is a real time computational tool for curvature analysis and the result is displayed as a color map on the preview of the modified surface.



It enables you to visualize the zones where the sign of curvature changes from concave to convex and vice-versa.


Improvement in color propagation

Improvements have been made in color management to provide better control over the application of colors. Now it is possible to do the following:

  • Set the color of a face different from the color of the solid.
    (Enhancement in the Edit Properties - General category.)

  • Assign component color.
    (New option in the Document Properties - General category.)

  • Inherit component/solid color.
    (New options in the Entity Properties - Drawing View - View Attributes category.)

  • Retain the color of the surfaces while using the Make Solid and Break Solid commands.
    (New Color option in the command's selection list.)

  • Define the color of the solid created by a feature.
    (New options in the System Options - Construction category.)

  • Retain the face colors while importing/exporting files using 3D converters.

Enhancement in the Environment command

The following enhancements have been made in the Environment command.

  • A new Cross option has been introduced in the Background and Reflection drop-down list.
    • When selected in the Background drop-down list, a 6-bit map image can be selected and visualized in the background.
    • When selected in the Reflection drop-down list, the selected image is reflected on to the model.

  • Under the Background node a new Size option enables you to regulate the size of the background. This option is available only when Sphere, Cube or Cross is selected in the Background drop-down list.

Enhancements in the Automatic Traceback Management

In version 2008.1, the automatic error traceback management was introduced as a quality initiative to receive customers' feedback to improve the quality of the product. The same has been significantly improved in this version.

Previously, the error reporting was managed only when explicitly activated by the user (using the Setting Wizard or from the Help menu commands). From this version it is possible to send the automatic logger information from the user system depending on the settings made in the Options/Properties dialog box.


Following are some of the significant enhancements that have been made in the Automatic Traceback Management:


  • A new Log File Management dialog box has been introduced to send the automatic logger information from your system. This dialog box appears on each application startup only when:
    — the option Show this dialog box at startup is selected
    — there is an unsent log and/or traceback

  • Henceforth, the system requirements include only an internet connection and need not have an email application running to send the tracebacks unlike earlier version.

  • The options of the new Log file management area in the System Options - General - Advanced category enable you to make settings for configuring the Automatic Logger to send information from your system. Also, the Error Report commands are no more available from the Help menu.

Note
The automatic logger information includes only the textual data referring to the calling sequence of a command leading up to the error or crash. This can make it easier for the programmer to identify and resolve the application bugs. It is important to note that no customer data is transferred through this improved traceback mechanism.

New Hide Inner Entities command

The new ViewHide Inner Entities command enables you to hide the entities inside the model/assembly that are not visible. The command is applied on those parts that are not visible from any of the predefined views. By hiding the entities, the hardware display list is reduced, which results in faster loading of the model with increase in the frame rate, hence faster refreshes.
You can hide different entities based on the selected level of precision and setting of other options in the selection list.


Enhancement in High Quality Rendering (HQR)

Starting from this version onwards, you can render a model with high quality rendering effects without the need to do any settings. When a model is opened in the application, start the ViewDisplayHigh Quality Shaded command to view the model rendered with a predefined environment and lighting settings.


Improvement in the Two-Entity Distance Map command

In this version the Max. distance option under More Options has been renamed to Max. presumed distance in the Two-Entity Distance Map selection list.
