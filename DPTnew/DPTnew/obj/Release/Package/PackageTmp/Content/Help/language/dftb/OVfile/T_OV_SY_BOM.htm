Symbolic expressions in a standard part list

Default Part Lists are ordinary ".e2" drawing files located in the "fmt\language" subfolder of the User configuration root folder (language is the installation language). Each list is described by two files:

  • A file with the prefix "h_" which describes the list header (h_bom.e2)



  • A file with the prefix "r_" which describes a row of the list (r_bom.e2)



When the list is filled in, the application appropriately positions the graphic symbols and converts all the source symbolic expressions that were used in the standard part list into the corresponding real value.


Symbolic expressions used in standard part lists

Here follows the description of all the symbolic expressions used in standard part lists. See the previous examples for default descriptions and date formats. For detailed information about the predefined properties used in part lists, see also the lists of Document properties, of Part properties and of Relation properties.

As the Part List is a group entity, any of the part properties that have to be displayed inside it have to be inserted in "$G" symbolic expressions.

$G{RELPOS}

Pos: Displays the position number.

$G{COMPNAME:f("#clear(<undef>@)")}

Part Number: Displays the part number. If the part number has a "<undef>" prefix, the whole part number will not be displayed.

$G{COMPDES}

Description: Displays the part description.

$G{COMPITEM}

IT: Displays the item type.

$G{COMPUMIS}

Unit: Displays the unit of measure.

$G{COMPQTY}

QTY: Displays the quantity.
