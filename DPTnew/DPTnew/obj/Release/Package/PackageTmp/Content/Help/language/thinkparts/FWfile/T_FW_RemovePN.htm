The Remove Part Number command enables you to delete the selected part from the thinkparts catalog.

You can find this command in the context menu that becomes available when you right-click on a part in an open catalog if a part number is assigned to the selected part.

When executed, the command deletes the selected entry in the thinkparts catalog. Note that if other entries are available, they are unaffected by this command. You cannot remove the original uncoded part entry, because the Remove Part Number command is not available in its context menu.

If the Remove files generated with Import command option is checked, Remove Part Number will also remove the cached model file from the Catalog X-Ref. folder in order to insure that the next time the part is imported it will have the correct part data assigned. If the option is not checked, the think3 application will import the existing part with the old part data.
