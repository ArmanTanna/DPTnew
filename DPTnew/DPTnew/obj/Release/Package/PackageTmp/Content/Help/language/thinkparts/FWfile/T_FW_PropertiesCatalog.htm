The Properties command, referred to an entity of type Catalog, enables you to manage its properties.

This command is located in the context menu available when right-clicking on a catalog of components listed in the browser frame and as a button in the Catalog Manager command dialog box.

When you start the command, it displays a dialog box with the following tabs:

General with the main catalog properties
Preview with the preview image of the catalog

The buttons at the bottom of the dialog have the following functions:

OK closes the dialog and saves the changes you have made to the thinkPLM database.
Cancel closes the dialog and discards the changes you have made.

General tab

This tab provides you with the following controls that enable you to inspect or set the catalog properties:

Catalog code: The name (maximum 32 characters) that identifies the catalog in the thinkPLM database. This field is mandatory when creating a new entity; it is grayed out (not modifiable) when editing the properties of an existing one.
Description: The descriptive name of the catalog. This value is displayed in the browser frame and actually becomes the name of the catalog. You are enabled to edit this field at your convenience.
Additional desc.: An additional field for the description of the catalog. You are enabled to edit this field at your convenience.
Future type: The entity type that will pertain to the part when it will be imported as a think3 component. The default setting of this field is COMPONENT, however, if you have created other entity types in your thinkPLM installation, you are enabled to use the button to select the appropriate one.
Folder: A read only field that reports the actual folder in your system where the files are stored. This is a subfolder of the folder specified in the Catalog root folder option.
