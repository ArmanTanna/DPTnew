Enhancements added to the Part List command

1.       Edit Part Properties command added to the context menu in the Part List Editor table

2.       Add custom properties shown as check boxes

3.       Differentiate the hidden and unhidden components rows in the Part List Editor table

4.       Restrict user to enter only specific data type while editing the cells in the Part List Editor table

 

 

Edit Part Properties command added to the context menu in Part List Editor table

It is now possible to  launch the  Edit Part Data command  from the context menu Edit Part Properties option of the Part List Editor table. It will be shown in the context menu upon selection of only one row.

On closure of the Part Data dialog, the part list is updated.

 

 

Add custom properties shown as check boxes

In previous versions custom data properties could be  of Integer, Text and Real data types and there was no option to add Boolean variable.

Now, if a new custom properties named like “anyname_BOOL” is added and its data type is Integer, we internally consider it as Boolean in the Part List Editor. 

This property will be shown as check-box in the part list editor. If the value of the property is 0, then the check box in Part List for that component will be unchecked, else checked.

Differentiate the hidden and unhidden components rows in Part List Editor table

In previous versions, there was no way in the part list dialog to differentiate the hidden ones from unhidden ones.

To help users identify such components, the text of the rows corresponding to the column will now be shown in Italics.

The context menu shows Hide Entities or Unhide Entities or both, depending on the components selected in the Part List.

 

 

Restrict users to enter only specific data type while editing the cells in Part List Editor

In previous versions, we allowed any text to be typed in the part list table if it was editable. We showed the data type error only on Apply. This could be confusing, since users could not know which value caused an error.

NOw, each column is a property and has a data type associated. We restrict users from entering any other data type in the column.

Ex:   RELQTY is of data type Real and hence doesn’t accept any Characters while typing.
