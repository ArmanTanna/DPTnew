Printing Issues

When trying to print out some parts of the online documentations, some issues may arise. Here are the work-arounds for such situations.

  • Printing help linked documents is not recursive
  • Printed help articles do not show the images
  • Printed help articles are not Complete
  • Printed help article does not look correct

Printing help linked documents is not recursive

Printing of the contextual help is possible, as documented in the relevant article. In the Print dialog box, a check box enables you to Print all linked documents.
Since the help system structure is made up of links intended for interactive use that enable you to browse back and forth throughout the articles without limitations, in order to avoid the possibility of endless print jobs the Print all linked documents is limited to the first nesting level.
In other words, if you print all documents linked by an article, only those directly linked to the latter will be printed; links present in the linked articles will be ignored.
Finally, please note that if any printed page should not look correct, you can print each of those pages one by one proceeding as described in "Printed help article does not look correct".

Printed help articles do not show the images

Depending on your operating system setup, on some Microsoft Windows configurations you may run into problems trying to print the help articles because the images do not show on paper.
This issue sometimes occurs when trying to print the article from an Internet browser window, such as the ones opened upon clicking on a link to another subject.
If this is your case, use the help system searching tools to locate the article that you wish to print, open it in the main help window, and print it by right-clicking in the right frame.

Printed help articles are not complete

When trying to print a document by right-clicking and selecting Print in the context menu, the printed document you actually obtain might be unexpectedly incomplete.
If this issue  — due to the internal structure of the document itself — occurs, please print the document again by right-clicking on the very top of the document itself (for instance, close to the heading) and choosing Print in that location. This action will be enough to solve the problem and obtain a complete printed copy of the document.

Printed help article does not look correct

Some pages in the online documentation are built at run-time (for example, this is the case of all the pages describing the Options/Properties categories). When you print one of such pages you may encounter one of the following issues:

  • Some parts are truncated.
  • Some images not visible on the screen are printed on the paper.

A valid work-around to solve this issue is to proceed as described in the following steps:

  • Maximize the document using the proper button on the upper right corner of its window.
  • Resize it once or twice.
  • Print it again.

In case the issue should still be there, repeat the above steps once more. The printed copy you obtain should now be correct.
