What's New in Service Pack 1

Version 2008.1 Service Pack 1 introduces the following improvements.

  • Enhancements in the Assign Properties command
  • Improvements in the Axis Generation Mechanism
  • New Chamfer command
  • Enhancements in the Surface Textures command
  • Enhancements in the entities' attributes in the drawing view
  • The new Die Design module
    • The new Fuzzy Offset command
    • The new GSM Spined Twist command

Enhancements in the Assign Properties command

The Assign Properties command has been functionally improved by opening by default the Dimension Properties toolbar, introduced with this version, instead of the Assign Properties dialog box.

The command selection list remains unchanged, so that by clicking the Properties node, the program displays the Assign Properties dialog box.


The command icon has been updated as follows:

Old icon New icon

The enhanced behavior of the command, moreover, enables you to first set the properties, and then select the entities to which you wish to assign them (post-selection). Until now, it was only possible to use the Dimension Properties toolbar to modify the properties of selected entities (pre-selection).


Improvements in the Axis Generation Mechanism

Significant enhancements have been made to improve the axis generation mechanism.

  • It is now possible to have the axes created automatically when a drawing is created from the current model. Also, all the axes created in the layout environment will be associative.

  • The axes belonging to non hidden entities are only visible.

  • Axis merging is possible for overlapping or partial-overlapping axes to create unique associative axes.

  • It is possible to create axes for both the front and top views (axis perpendicular or parallel to the views).

  • To hide the automatically generated axes, you can apply the Hide Dimensions Axes Edges command.

  • Axis extension is now allowed, without any of the previous limitations.

  • The new option - Add axes to rotational surfaces - has been introduced in the Entity Properties - Drawing View - Visibility category replacing the old Hidden line+axis together with Add axes to holes and rotational slots. The removed options (used to create unassociated axes on a drawing view) are available only when editing old drawing views using them; in such a case the geometry representation item will be presented as Hidden Lines (ver. < 2008T1-SP1).

    When you open a drawing file created in earlier versions and in it:
    • if the Add axes to holes and rotational slots option is selected, the geometry representation Hidden Lines (ver. < 2008T1-SP1) will be available in the drop-down list.
    • if the Add axes to holes and rotational slots option is not selected, the Hidden Lines (ver. < 2008T1-SP1) option will not be available in the interface.
    • if the user unchecks the Add axes to holes and rotational slots option, the program automatically switches the geometry representation to Hidden lines mode and the Hidden Lines (ver. < 2008T1-SP1) option will no more be available.


  • Activating the Add axes to rotational surfaces option disables all the other axes types creation (Datum axes, Profile references, Centerline curves); before creating the new axes removes the existing ones from the view.

New Chamfer command

The following significant enhancements have been made in the dimensioning of chamfer:


  • Till 2008.1 Service Pack 1, chamfer dimensioning was only possible through Smart Dimension command. Now it is possible to use the new InsertDimensionChamfer command, which enables you to quickly insert a chamfer dimension by selecting a line representing the chamfer edge. For further details, refer to the online help.

  • In order to include 45° chamfer to the chamfer dimension, a new linear dimension type with a specific display list and a parameter set to define its representation is needed. As a consequence, the following style specializations have been introduced in this version:

    • The new dimension type, 45° Chamfer dimension, in the Specialize Style dialog box
    • The new options, Aspect and Symbol, in the 45° Chamfer area of the Entity Properties - Dimension - Line and Arrow - Advanced and Entity Properties - Dimension - Symbol categories.

Enhancements in the entities' attributes in the drawing view

Till previous version, the profile entities did not have, in 2D environment, their own attributes. Starting from this version, it is possible to manage different attributes for entities belonging to a profile. Also, the width management has been refined.


The following improvements have been made to the behavior of the entity attributes management:

  • The old Keep model geometry color and layer option inside Entity Properties - Drawing View - View Attributes category, which managed the model entities' attributes when creating a drawing view with wireframe visibility option, has been renamed to Keep model attributes.

  • If Keep model attributes is selected and the Representation option in Drawing View - Visibility is set to:
    • Wireframe — the model's color, layer, linetype and width will be retained and hence the Color, Layer, Width and Line Type options are grayed out. (In earlier versions, these attributes were still enabled.)

    • Hidden lines — only the model's color and layer attributes will be retained and therefore only the Width and Line Type options are enabled. (In earlier versions, all the attributes were enabled.)

The new Die Design module

The new Die Design module has been introduced in this version that constitutes of following three commands, which have been developed specifically to facilitate Die designing.

  • InsertCurveCutting Line (this command is available from version 2008.1)
  • InsertSurfaceFuzzy Offset
  • ModifyGlobal Shape ModelingSpined Twist

All the above three commands are also present in the Die Design toolbar.

The new Fuzzy Offset command

The new Fuzzy Offset command generates an offset surface which is not a precise offset but has blurry resemblance to the original surface. This type of offset surface is useful for creating the back side of the die/punch head.


The parent skin The resulting fuzzy offset surface

The program performs the following steps to develop a fuzzy offset surface:

  1. Extract some sampling points from the parent skin
  2. Develop a new surface from these points

On starting the command, you need to select a solid skin. The skin represents the actual profile of the die head. Thereafter, specify the offset value and offset direction. When you click the Preview button, the program generates the offset surface and shows the preview. Now you can perform a distance analysis to see the percentage deviation of points from their true offset location. If satisfactory, you can apply the command; otherwise modify the U and V parameters of the offset surface or the number of iterations to get a better point distribution.


The new GSM Spined Twist command

The Spined Twist is a new GSM command that can help in compensating those parts in which the spring-back effect generates a twist. Using this command you can provide a twist to a part along a predefined spine. It is also possible to define the degree of twist at various sections of the part.


After starting the command you need to select the surfaces to be twisted and a spine curve about which the twist will take place. As an example consider the part below.



Upon making the selections, sections will be displayed over the spine. The command gives you full control over the number and location of the sections and the possibility to specify the twist angle value at each section. You can also control the preview of the sections and define the stamping direction.



After visualizing the preview of the twisted surface, and if satisfactory, you can apply the command. The program generates the twist as per an interpolation twist-angle-law based on the angle values specified at various sections. The twisted surface will look like the one shown below.
