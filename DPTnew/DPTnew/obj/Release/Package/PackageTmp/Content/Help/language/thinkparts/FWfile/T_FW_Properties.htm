The Properties command, referred to an entity of type Index, enables you to manage its properties.

This command is located in the context menu available when right-clicking on an index listed in the browser frame and as a button in the Catalog Manager command dialog box.

When you start the command, it displays a dialog box with the following tabs:

General with the main index properties
Preview with the preview image of the index

The buttons at the bottom of the dialog have the following functions:

OK closes the dialog and saves the changes you have made to the thinkPLM database.
Cancel closes the dialog and discards the changes you have made.

General tab

This tab provides you with the following controls that enable you to inspect or set the index properties:

Index code: The name (maximum 32 characters) that identifies the index in the thinkPLM database. This field is mandatory when creating a new entity; it is grayed out (not modifiable) when editing the properties of an existing one.
Description: The descriptive name of the index as it appears in the browser frame. You are enabled to edit this field at your convenience.
Folder: A read only field that reports the actual folder in your system where the files are stored. This is a subfolder of the folder specified in the Catalog root folder option.
