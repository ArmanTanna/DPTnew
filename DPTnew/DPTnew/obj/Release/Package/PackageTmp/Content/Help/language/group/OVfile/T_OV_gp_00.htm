What is a Group?

In the 2D environment, it is often useful to be able to treat a set of entities as a single entity but without losing the possibility of deleting, inserting or editing individual entities in the set.

In the think3 application, this facility is provided by Groups. A Group is treated as a single entity identified by a name and a point of origin. It can contain entities of all types supported by the program, including other groups, thus enabling you to create a hierarchical structure.

Typically, groups are entities of two-dimensional drawing environments and have the following advantages:
 

  • They make it easier to manage sets of entities
  • They enable you to represent properties common to two or more entities
  • They confer modularity on a drawing
  • They give a drawing a tree structure.

A frame on the left of the graphics area, called the Drawing Structure, displays the group structure.

 

Non-Shared Groups and Shared Groups

When you create a group (see New Group) you can decide whether your group will be shared or not. The main differences are the following:
 

  • Once a shared group has been created and set as current (see Set Current Group), if you edit it and modify it, all the instances of that group will be modified as well, differently from what happens with non-shared groups.
  • A non-shared group has no relations with other non-shared groups. Thus, non-shared groups with the same name may consist of different sets of entities. On the contrary, shared groups with the same name consist of the same set of entities.

Now, suppose you created a shared group representing a part in your drawing. You can create an external file for that part (external reference, or X-Reference (groups)) or you can leave it in your drawing as local. Advantages and disadvantages are described in the following section.

A specific tab in the frame on the left of the graphics area displays the list of shared groups (see Shared Group List).

When applying the Info commands to entities belonging to Shared Groups, specific data are displayed as described in "Specific output data of the Info commands when applied to entities belonging to shared groups"

 

Group X-Reference

In this case the shared group is a program's 2D file and the drawing contains a link to this file.

The following are the advantages:
 

  • Any changes made to the external file containing the shared group representing the part are automatically reproduced in the drawing and in all the drawings where an X-Reference (groups) to that part is contained.
  • Since the drawing only contains a link to an external file and not the entity itself, the size of the file will not be increased. Furthermore, when using the same X-Reference group in several drawing the total size will not be affected, no matter how many times it is used.

And these are the disadvantages:
 

  • Before making any changes to an external reference group you must verify how many drawings are affected by the changes. The change you are going to make might actually be appropriate for the current drawing, but not for other drawings containing the same X-Reference (groups).

Anyway, you can turn an X-Reference group into a local shared group at any time (see the Unlink Shared Group command).

 

Local Shared Groups

In this case the shared group and the entities that it contains are stored directly in the drawing.

These are the advantages:

  • You do not need to create a new file for each part.
  • You can group one or more shared group in a single local shared group inside the drawing so that a different hierarchy structure is defined. Using the local shared group as an aggregation of shared groups, you can define, for the current drawing only, a particular hierarchy structure in order to reproduce or comply with the final product structure.

These are the disadvantages:

  • In order to modify a part represented as a local shared group you must open the drawing where such shared group is used.
  • Entities used inside the local shared group are stored in the drawing itself. This means that the size of the drawing is affected by the size of these entities. If the shared group is used inside several drawings, the size increases accordingly.
  • If the same shared group is used as local component in several drawings, individual instances are not connected between themselves and changes made on one of them will have no impact on the others (in some cases this might be considered an advantage).

You should therefore manage as local shared groups all those objects which have been specifically designed for a drawing and that will not be reasonably used in other drawings.

Furthermore, local shared groups should be used to define a local hierarchy structure respecting the nature of the product. Using X-References (groups) would require a set of files for each possible product configuration thus increasing the total number of files.

Local Shared Groups are persistent
Please note that the definition or archetype of a local shared group is still availbale though you may have deleted all of its copies in the drawing. Specific Commands to remove archtypes of local shared groups with no copies in the drawing are available.

You can always turn a local shared group into an X-Reference (groups) by using the Make X-Reference Group command.

 

Group Hierarchy

The possibility of nesting groups inside one another creates a tree structure.

The illustration above shows the structure of a group (Parent) containing two other groups (Child #1 and Child #2).

The relationships between the different levels in the structure can be defined as follows:

  • PARENT: the parent of a group is the group that is one level higher than it and contains it
  • CHILD: the child of a group is a group that is one level lower than it and is contained by it
  • BROTHER: the brother of a group is a group at the same level as it, inside the same parent group.

The drawing as a whole is considered to be a group at the highest level which contains all the others. This corresponds to nesting level 0 because the drawing cannot be contained in another group.

See Drawing Structure for further details.

 

Logical States of Non-Shared Groups

At any time, every non-shared group has one of the following logical states specifying the type of actions that you can apply to it:

  • CLOSED: The group is treated as a single entity. You can therefore delete, move and copy it like any other entity but you cannot act on single entities inside it
  • OPEN: When a group is open, you can edit or delete single entities inside it. To open a group, you must first open the groups at all the levels higher than it, if any. The drawing, as a group, is always open.

There may be different groups in the open or closed state at the same time.

When you save the entire drawing as a file, any open groups are automatically closed. Immediately after being created, a group is in the closed state; similarly, when you load or insert a drawing, all the groups inside it are in the closed state.
You can open a group by clicking on the corresponding icon in the tree structure of the drawing:

About opening and closing non-shared groups, see the Open Group and Close Group commands.

 

Current Group

To avoid ambiguity and confusion, the program allows you to add new entities only to one group at a time.

The single group to which new entities are added is called the Current Group. You can choose the Current Group at any time (see the Set Current Group command).

All new entities created are assigned to the Current Group (which may be the drawing as a whole as the parent group at the highest level; see the Top Level Group command)

 

Breaking a Group

You can break a group out into its single component entities. Any child groups remain unchanged as groups but move up one level in the tree structure, that is, they change parent.

To break a group into its component entities, use the Break command.

 

Inserting groups into the drawing

The think3 application provides several methods to insert groups into a drawing.
 

  • To insert an X-Reference group, use the Group Cross Reference command
  • To add copies of existing shared groups, right-click on the group in the drawing or on the icon in the Drawing Structure and select InsertGroup Copy. You can also right-click on the icon in the document Shared Group list tab and select Insert.
  • To insert any type of group, use the Insert command in the Group Library.

 

Editing a group

There are several methods to edit groups.

  • To edit non-shared groups just by modifying and/or removing entities (but not adding), open the group and make your changes. If you need to add entities, set the group as current by using Set Current Group or simply by double-clicking on the group (which also sets the group as the current one).
  • To edit shared groups, right-click on the group in the drawing or on the icon in the Drawing Structure and select Start Local Edit in the context menu. You can also right-click on the icon in the document Shared Group list tab and select Start Local Edit.
    Alternatively, you can set the group as current with the Set Current Group command, which applies also to shared groups.
    You can set the default color mode for shared group editing as described in "Setting the Default Color Mode for Shared Group Editing".

 

Drawing Views as Groups

Drawing views created automatically from a 3D model are particular non-shared groups. Except from not being enabled to edit the entities that are automatically inserted by the program at creation time, views can be regarded as non-shared groups. You can open and close them, set them as current, add entities to them.

 

How entities belonging to a group are displayed

The colors used by the think3 application to display groups and the entities belonging to groups can be defined in the Group color area of the Drawing Group-Color category of the Document Properties.

 

Group libraries

A group library is a collection of groups which you can use to import frequently used groups into your drawing. A Group Library is simply a folder in which each group is contained in a ".e2" file. See the description of the Group Library command for further details.
