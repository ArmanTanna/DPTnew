ModifyProfileConvert

To convert the profiles created using the old version of the program (prior to the release 2006.2) you can use the Convert Profile command.

Old profiles (created prior to the release 2006.2) can be converted only by entering the profile edit mode. However, to facilitate the users who need to change their legacy profile data without entering the profile environment, the Convert Profile command can be used. For example, such a situation arises while using commands such as the Move/Copy to Current Work Plane command (ModifyProfileMove/Copy to Current Work Plane) where you can select and perform operations on a profile while being inside the model environment. Under these circumstances, you can use the Convert Profile command.

To use this explicit conversion of profiles, start the command and select the old profile that you wish to convert to the new version (release 2006.2 and later). Note that you can only select the profile made using an older version of this application.

Also be aware that if your old profile does not require specifically to be upgraded to the new profile format, unless you expect some profile level changes in your work. You are advised NOT to use this command explicitly to perform a conversion, unless the situation warrants it.

Note

There is a profile conversion policy put into practice. If your legacy models have to be rebuilt, and if their profiles are not changed during the process, the application will NOT update the old profile data in the file. On the other hand, if the changes are to be made at a profile level, the existing profiles in those files will be automatically converted according to the new profile environment standards. Note that these conversions are automatically performed by the program and are totally transparent to the users. When a conversion is performed a warning message is shown to the user.
