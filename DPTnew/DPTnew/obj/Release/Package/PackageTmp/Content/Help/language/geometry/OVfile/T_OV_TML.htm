think3 application Text Markup Language (TML)

TML (the think3 application Text Markup Language) is an XML-like — and XML-compliant — text markup language provided with the program in order to support and create multiline texts (*). The following sections explain how to create multiline texts using TML formatting.

  • TML Tags

Foreword

Text will be recognized as formatted multiline text if it starts with the <TEXT version="1.0"> tag, and it ends with the </TEXT> tag. Any text without the starting <TEXT version="1.0"> will be treated as plain unformatted text.

Once you have started your formatted text by typing the <TEXT version="1.0"> tag, you can go on writing it using the available TML tags.

  • A tag is a code interpreted by the program as a text formatting instruction. All TML tags are enclosed in angular brackets: "<TAG ..>".
  • Tags can have attributes. An attribute specifies additional information concerning the desired behavior of the chosen tag and is to be specified inside the tag in the form: attribute="value". For example in:
    <FONT face="Arial">
    the "face" attribute specifies which type of characters is to be used in the subsequent text.
  • To properly define the portion of text to be formatted, a closing tag must be inserted just after the text to be formatted in a certain way. For example, if a certain string is to be displayed using the Courier characters, you will have to write:
    <FONT face="Courier">text to be written in Courier</FONT>.
    The <BR/> and <SP/> tags (see below) do not need a closing instruction.
    Graphically, you can appreciate the difference as tags not needing a corresponding closing tag end with a slash (<BR/>, <SP/>), while all opening tags do not have any slash (for example: <FONT ...>) and all closing tags begin with a slash rather than ending with it (for example: </FONT>).
  • Tags can be nested in order to obtain proper formatting. The sequence of opening/closing tags must be XML-compliant. In other words, the last tag opened before the text must be closed for the first. The following sequence is correct:
    <B><I>Correct Text: bold and italic</I></B>
    The following is incorrect:
    <B><I>Correct Text: bold and italic</B></I>
    To avoid confusing tag nesting, the following sequence:
    <B> Let's <I> try </B> again </I>
    Must be rewritten as follows:
    <B> Let's <I> try </I></B><I> again </I>

About white spaces
TML adopts HTML white space policy:
  • All TABs and line breaks are ignored.
  • Continuous blank spaces are shrinked to a single space. The first encountered is kept (spaces with different font format have different width). All blanks and tabs at the beginning of a new line are ignored.

<SR> ... </SR> 		Enables you to create orderly piles of rows, which can also be nested. The <STACK> tag defines a section within which each <SR> tag is a row. Using the separator attribute, you can choose the row separator:
none
line
slash
Height of texts within <SR> tags is by default scaled by a 0.7 factor. You can define the scale of texts in rows by using the attribute scale="n" where n is a non negative floating value. Some important notes:
  • <STACK> tags can contain only <SR> tags.
  • <SR> tags cannot contain <NOWRAP> and <BR/> tags: line wrapping is disabled in stack rows.
  • <STACK> tags can be nested.

<VALIGN type="center">
Here is a pile of rows:
<STACK separator="line" scale="1">
   <SR>Row1</SR>
   <SR>Row2</SR>
   <SR>Row3</SR>
</STACK>
</VALIGN>
</TEXT>
