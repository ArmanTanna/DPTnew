Annotations, in their simplest form, are text labels tied to the model with a leader. Unlike traditional labels, annotations may be viewed in 3D and float around the "Point of Interest" — the point on the model the Annotation is anchored to — as the view orientation changes. Furthermore, these notes can expand (see also "Expanded Mode" in the glossary) to convey information in several multimedia formats (such as HTML, AVI, Microsoft® Word Doc, etc.).

1 Point of Interest

The objectives of annotations are:

  • Easily attach metadata to the CAD model
  • Allow easy navigation amongst the new imbedded data
  • Reduce the need to create 2D drawings in order to attach metadata to the part design.

The annotation also contains storage for keywords and category strings. This text along with the label note and content will allow the user to browse or filter the annotations being displayed at any given time. The user Interface provides a browse string and a tree organizing annotations by category.

Annotations also comply with the project browser mechanism allowing any externally referenced annotation data to be managed by thinkPLM.
